## 1. Introduction

The 65C02 shares many similarities with the NMOS 6502 but includes additional instructions, addressing modes, and variations. This guide documents these differences systematically.

**Table Column Definitions:**
- **OP**: Opcode in hexadecimal
- **LEN**: Instruction length in bytes
- **CYC**: Number of cycles required
- **MODE**: Addressing mode
- **FLAGS**: Affected status flags
- **SYNTAX**: Assembler syntax example

Lowercase letters in the CYC column indicate footnotes describing conditional cycle variations.

## 2. 6502 Instructions with Additional Addressing Modes

### ADC AND CMP EOR LDA ORA SBC STA - (zp) Addressing Mode

| OP | LEN | CYC | MODE | FLAGS      | SYNTAX       |
|----|-----|-----|------|------------|--------------|
| 72 | 2   | 5 a | (zp) | NV....ZC   | ADC ($12)    |
| 32 | 2   | 5   | (zp) | N.....Z.   | AND ($12)    |
| D2 | 2   | 5   | (zp) | N.....ZC   | CMP ($12)    |
| 52 | 2   | 5   | (zp) | N.....Z.   | EOR ($12)    |
| B2 | 2   | 5   | (zp) | N.....Z.   | LDA ($12)    |
| 12 | 2   | 5   | (zp) | N.....Z.   | ORA ($12)    |
| F2 | 2   | 5 a | (zp) | NV....ZC   | SBC ($12)    |
| 92 | 2   | 5   | (zp) | ........   | STA ($12)    |

**Note a)**: 6 cycles if the D flag is set (decimal mode)

The new (zp) addressing mode provides a more efficient alternative to (zp),Y when Y is zero. Key distinctions include:

- STA (zp) executes in 5 cycles versus 6 cycles for STA (zp),Y
- ADC and SBC require 6 cycles in decimal mode instead of 5
- Both (zp) and (zp,X) modes are functionally identical when X is zero, but "(zp)" is one cycle faster
- Flag effects remain consistent across all addressing modes for each instruction

### BIT - imm, abs,X, zp,X Addressing Modes

| OP | LEN | CYC | MODE   | FLAGS      | SYNTAX        |
|----|-----|-----|--------|------------|---------------|
| 89 | 2   | 2   | imm    | ......Z.   | BIT #$12      |
| 34 | 2   | 4   | zp,X   | NV....Z.   | BIT $34,X     |
| 3C | 3   | 4 a | abs,X  | NV....Z.   | BIT $5678,X   |

**Note a)**: 5 cycles if indexing across a page boundary

BIT gains three new addressing modes. Notably, the immediate mode affects only the Z flag, whereas abs and abs,X modes affect N, V, and Z flags. This represents "the only instruction in the 6502 family that does not affect the same flags in all of its available" modes. The Z flag result derives from an AND operation between the accumulator and operand, but the accumulator remains unchanged.

Example behavior difference: BIT $12 (absolute) clears the V flag if memory contains $00, while BIT #$00 (immediate) does not affect V.

### DEC INC - acc Addressing Mode

| OP | LEN | CYC | MODE | FLAGS      | SYNTAX    |
|----|-----|-----|------|------------|-----------|
| 3A | 1   | 2   | acc  | N.....Z.   | DEC       |
| 1A | 1   | 2   | acc  | N.....Z.   | INC       |

DEC and INC (without operands) decrement or increment the accumulator respectively, similar to DEX, DEY, INX, and INY but operating on the A register. Cycle counts match other implied-mode register operations.

Some assemblers permit alternative syntax:
- DEC: `DEA` or `DEC A`
- INC: `INA` or `INC A`

### JMP - (abs,X) Addressing Mode

| OP | LEN | CYC | MODE    | FLAGS      | SYNTAX         |
|----|-----|-----|---------|------------|-----------------|
| 7C | 3   | 6   | (abs,X) | ........   | JMP ($1234,X)   |

The X register adds to the absolute address, with the result containing the jump destination (low byte first). This facilitates efficient jump table implementations.

**6502 Jump Table Example:**
```assembly
; Enter with: routine number (0 to 2) in accumulator
;
       ASL
       TAX
       LDA TABLE+1,X
       PHA
       LDA TABLE,X
       PHA
       RTS
TABLE: DW  ROUTINE0-1,ROUTINE1-1,ROUTINE2-1  ; Note the -1
```

**65C02 Equivalent (More Efficient):**
```assembly
; Enter with: routine number (0 to 2) in accumulator
;
       ASL
       TAX
       JMP (TABLE,X)
TABLE: DW  ROUTINE0,ROUTINE1,ROUTINE2  ; note: no -1
```

**Alternative 6502 Jump Table:**
```assembly
; Enter with: routine number (0 to 2) in X
;
        LDA TABLEH,X
        PHA
        LDA TABLEL,X
        PHA
        RTS
TABLEH: DB  >ROUTINE0-1,>ROUTINE1-1,>ROUTINE2-1  ; high bytes
TABLEL: DB  ROUTINE0-1,ROUTINE1-1,ROUTINE2-1     ; low bytes
```

**65C02 Equivalent (Single Table):**
```assembly
; Enter with: routine number times 2 (0, 2, or 4) in X
;
       JMP (TABLE,X)
TABLE: DW  ROUTINE0,ROUTINE1,ROUTINE2  ; note: no -1
```

The 65C02 version supports up to 128 routine addresses in a single table, while the 6502 version accommodates 256 using two separate tables.

**Extended 65C02 Example (Up to 256 Addresses):**
```assembly
; Enter with routine number (0 to 255) in accumulator
;
       ASL
       TAX
       BCS HIGH
       JMP (TABLE,X)
HIGH:  JMP (TABLE+128,X)
TABLE: DW  ROUTINE0,ROUTINE1,ROUTINE2,ROUTINE3
       ; etc.
       DW  ROUTINE252,ROUTINE253,ROUTINE254,ROUTINE255
```

## 3. Additional Instructions

### BRA - Branch Always

| OP | LEN | CYC | MODE | FLAGS      | SYNTAX        |
|----|-----|-----|------|------------|---------------|
| 80 | 2   | 3 a | rel  | ........   | BRA LABEL     |

**Note a)**: 4 cycles if the branch is to a different page

BRA executes unconditionally, unlike conditional branch instructions. The destination address must fall within -128 to +127 bytes of the next instruction following BRA. Cycle counts match other branch instructions: 3 cycles normally, 4 when crossing a page boundary.

### PHX PHY PLX PLY - Push/Pull X or Y Register

| OP | LEN | CYC | MODE | FLAGS      | SYNTAX    |
|----|-----|-----|------|------------|-----------|
| DA | 1   | 3   | imp  | ........   | PHX       |
| 5A | 1   | 3   | imp  | ........   | PHY       |
| FA | 1   | 4   | imp  | N.....Z.   | PLX       |
| 7A | 1   | 4   | imp  | N.....Z.   | PLY       |

These instructions push or pull the X or Y registers to/from the stack, eliminating the need for transfer instructions (TXA, TAX, TYA, TAY) that affect the accumulator. PLX and PLY set the N and Z flags based on the pulled value. Cycle counts match PHA and PLA equivalents.

### STZ - Store Zero

| OP | LEN | CYC | MODE  | FLAGS      | SYNTAX        |
|----|-----|-----|-------|------------|---------------|
| 64 | 2   | 3   | zp    | ........   | STZ $12       |
| 74 | 2   | 4   | zp,X  | ........   | STZ $12,X     |
| 9C | 3   | 4   | abs   | ........   | STZ $3456     |
| 9E | 3   | 5   | abs,X | ........   | STZ $3456,X   |

STZ stores $00 (zero) at the specified memory location without requiring the accumulator to be cleared first. Functionally equivalent to `LDA #$00` followed by `STA`, but eliminates accumulator manipulation. Cycle counts match STA across all addressing modes.

Performance note: STZ offers minimal improvement in typical scenarios. For example, clearing a page of memory:

```assembly
     LDX #$00
     TXA
LOOP STA $1000,X
     INX
     BNE LOOP
```

Replacing STA with STZ and removing TXA yields only marginal gains: one fewer byte and two fewer cycles total (2561 vs. 2563 cycles; 2817 vs. 2819 if BNE crosses a page boundary).

### TRB - Test and Reset Bits

| OP | LEN | CYC | MODE | FLAGS      | SYNTAX        |
|----|-----|-----|------|------------|---------------|
| 14 | 2   | 5   | zp   | ......Z.   | TRB $12       |
| 1C | 3   | 6   | abs  | ......Z.   | TRB $3456     |

TRB performs two distinct operations simultaneously:

**Z Flag Effect**: Determined by bitwise AND of the accumulator with the memory operand. The accumulator remains unmodified. This matches the BIT instruction's flag behavior.

**Memory Effect**: Bits set to 1 in the accumulator are cleared (reset) in memory; bits set to 0 in the accumulator leave memory unaffected. This is mathematically equivalent to: `memory AND (accumulator XOR $FF)`

Functional equivalent code:
```assembly
BIT operand
PHP
PHA
EOR #$FF
AND operand
STA operand
PLA
PLP
```

Except TRB preserves N and V flags and requires no stack space.

**Example 1:**
```assembly
LDA #$A6
STA $00
LDA #$33
TRB $00
; Result: Z=0 ($A6 AND $33 = $22)
;         $00 contains $84 ($A6 AND $CC = $84)
;         Accumulator=$33 (unchanged)
```

**Example 2:**
```assembly
LDA #$A6
STA $00
LDA #$41
TRB $00
; Result: Z=1 ($A6 AND $41 = $00)
;         $00 contains $A6 (unchanged, since ANDing with $BE = no change)
;         Accumulator=$41 (unchanged)
```

Note: When Z=1 after TRB, memory retains its original value because any bit set in the accumulator must have been clear in memory.

### TSB - Test and Set Bits

| OP | LEN | CYC | MODE | FLAGS      | SYNTAX        |
|----|-----|-----|------|------------|---------------|
| 04 | 2   | 5   | zp   | ......Z.   | TSB $12       |
| 0C | 3   | 6   | abs  | ......Z.   | TSB $3456     |

TSB performs two distinct operations simultaneously:

**Z Flag Effect**: Determined by bitwise AND of the accumulator with the memory operand, identical to BIT and TRB. The accumulator remains unmodified.

**Memory Effect**: Bits set to 1 in the accumulator are set to 1 in memory; bits set to 0 in the accumulator leave memory unaffected. This is mathematically equivalent to: `memory OR accumulator`

Functional equivalent code:
```assembly
BIT operand
PHP
PHA
ORA operand
STA operand
PLA
PLP
```

Except TSB preserves N and V flags and requires no stack space.

**Example 1:**
```assembly
LDA #$A6
STA $00
LDA #$33
TSB $00
; Result: Z=0 ($A6 AND $33 = $22)
;         $00 contains $B7 ($A6 OR $33 = $B7)
;         Accumulator=$33 (unchanged)
```

**Example 2:**
```assembly
LDA #$A6
STA $00
LDA #$41
TSB $00
; Result: Z=1 ($A6 AND $41 = $00)
;         $00 contains $E7 ($A6 OR $41 = $E7)
;         Accumulator=$41 (unchanged)
```

## 4. Additional Rockwell 65C02 and WDC 65C02 Instructions

### BBR BBS - Branch on Bit Reset or Set

| OP | LEN | CYC | MODE   | FLAGS      | SYNTAX            |
|----|-----|-----|--------|------------|-------------------|
| 0F | 3   | 5 a | zp,rel | ........   | BBR0 $12,LABEL    |
| 1F | 3   | 5 a | zp,rel | ........   | BBR1 $12,LABEL    |
| 2F | 3   | 5 a | zp,rel | ........   | BBR2 $12,LABEL    |
| 3F | 3   | 5 a | zp,rel | ........   | BBR3 $12,LABEL    |
| 4F | 3   | 5 a | zp,rel | ........   | BBR4 $12,LABEL    |
| 5F | 3   | 5 a | zp,rel | ........   | BBR5 $12,LABEL    |
| 6F | 3   | 5 a | zp,rel | ........   | BBR6 $12,LABEL    |
| 7F | 3   | 5 a | zp,rel | ........   | BBR7 $12,LABEL    |
| 8F | 3   | 5 a | zp,rel | ........   | BBS0 $12,LABEL    |
| 9F | 3   | 5 a | zp,rel | ........   | BBS1 $12,LABEL    |
| AF | 3   | 5 a | zp,rel | ........   | BBS2 $12,LABEL    |
| BF | 3   | 5 a | zp,rel | ........   | BBS3 $12,LABEL    |
| CF | 3   | 5 a | zp,rel | ........   | BBS4 $12,LABEL    |
| DF | 3   | 5 a | zp,rel | ........   | BBS5 $12,LABEL    |
| EF | 3   | 5 a | zp,rel | ........   | BBS6 $12,LABEL    |
| FF | 3   | 5 a | zp,rel | ........   | BBS7 $12,LABEL    |

**Note a)**: 1 additional cycle if branch taken within same page; 2 additional cycles if branch taken to different page

BBR tests a specified zero page location and branches if the bit is clear (BBR) or set (BBS). Unlike other branch instructions, BBR and BBS take constant 5 cycles regardless of branch outcome, distinguishing them from conditional branches that vary based on condition.

**Advantages:**
- Test any of 8 bits without affecting flags or registers
- Particularly useful for testing bit 0 (even/odd) or software flags

**Limitations:**
- Single addressing mode (zero page only; no X/Y indexing)
- Restricted to zero page memory addresses, potentially complicating I/O mapping

**Syntax Variations:**

Standard form (bit specified in mnemonic):
```assembly
BBR0 $12,LABEL
```

Alternative form (bit as operand, allowing numeric expressions):
```assembly
FLAGS   EQU $AB
VERBOSE EQU 3                   ; verbose flag is bit 3

        BBR VERBOSE,FLAGS,LABEL ; skip if not verbose
```

The instruction encoding places the zero page address in byte 2 and the signed branch displacement in byte 3.

### RMB SMB - Reset or Set Memory Bit

| OP | LEN | CYC | MODE | FLAGS      | SYNTAX      |
|----|-----|-----|------|------------|-------------|
| 07 | 2   | 5   | zp   | ........   | RMB0 $12    |
| 17 | 2   | 5   | zp   | ........   | RMB1 $12    |
| 27 | 2   | 5   | zp   | ........   | RMB2 $12    |
| 37 | 2   | 5   | zp   | ........   | RMB3 $12    |
| 47 | 2   | 5   | zp   | ........   | RMB4 $12    |
| 57 | 2   | 5   | zp   | ........   | RMB5 $12    |
| 67 | 2   | 5   | zp   | ........   | RMB6 $12    |
| 77 | 2   | 5   | zp   | ........   | RMB7 $12    |
| 87 | 2   | 5   | zp   | ........   | SMB0 $12    |
| 97 | 2   | 5   | zp   | ........   | SMB1 $12    |
| A7 | 2   | 5   | zp   | ........   | SMB2 $12    |
| B7 | 2   | 5   | zp   | ........   | SMB3 $12    |
| C7 | 2   | 5   | zp   | ........   | SMB4 $12    |
| D7 | 2   | 5   | zp   | ........   | SMB5 $12    |
| E7 | 2   | 5   | zp   | ........   | SMB6 $12    |
| F7 | 2   | 5   | zp   | ........   | SMB7 $12    |

RMB clears (resets) a specified bit in a zero page location; SMB sets a specified bit to 1. These instructions are commonly paired with BBR and BBS for bit manipulation.

**Relationship to TRB/TSB:**

RMB and SMB function similarly to TRB and TSB respectively, but with key differences:

- RMB/SMB manipulate only a single bit; TRB/TSB can affect multiple bits
- RMB/SMB support only zero page addressing; TRB/TSB support zero page and absolute
- RMB/SMB do not use the accumulator, leaving it available
- RMB/SMB do not affect any flags (TRB/TSB affect the Z flag)

**Practical Consideration:** Since preserving the Z flag is rarely necessary (unlike other flags), RMB/SMB offer limited advantage over TRB/TSB in most scenarios.

**Syntax Variations:**

Standard form (bit specified in mnemonic):
```assembly
RMB0 $12
```

Alternative form (bit as operand):
```assembly
RMB 0,$12
```

The second syntax permits bit specification via numeric expressions:
```assembly
FLAGS   EQU $AB
VERBOSE EQU 3

        RMB VERBOSE,FLAGS
```

## 5. Additional WDC 65C02 Instructions

### STP - Stop the Processor

| OP | LEN | CYC | MODE | FLAGS      | SYNTAX    |
|----|-----|-----|------|------------|-----------|
| DB | 1   | 3   | imp  | ........   | STP       |

STP halts the 65C02 by stopping the clock input, transitioning the processor into a low-power state. Execution resumes only upon a hardware reset (RES pin activation).

**Use Cases:**
- Low-power applications requiring minimal energy consumption
- Battery-powered systems in sleep mode

**Limitations:** Rarely used in typical applications due to the drastic nature of the halt (no resumption without reset).

### WAI - Wait for Interrupt

| OP | LEN | CYC | MODE | FLAGS      | SYNTAX    |
|----|-----|-----|------|------------|-----------|
| CB | 1   | 3   | imp  | ........   | WAI       |

WAI places the 65C02 into a low-power sleep state, awaiting a hardware interrupt (IRQ, NMI, or RESET).

**Benefits:**

1. **Power Reduction:** Minimizes power consumption during idle periods
2. **Interrupt Responsiveness:** Guarantees immediate interrupt recognition after the third cycle completes

**Normal Interrupt Behavior:**
When an interrupt occurs mid-instruction (e.g., during ADC execution), the processor must complete the current instruction before acknowledging the interrupt and jumping to the interrupt vector.

**WAI Behavior:**
Once WAI's third cycle completes, the processor waits for an interrupt and responds without additional delay.

**IRQ with Interrupt Disable Flag (I flag set):**

When the I flag is set (interrupts disabled), WAI offers unique functionality:
- Upon IRQ occurrence, the processor continues with the next instruction instead of jumping to the interrupt vector
- The interrupt handler becomes inline code rather than a separate subroutine
- The interrupt completes within a single cycle
- No RTI (return from interrupt) is required, reducing cycle overhead

This enables inline interrupt handling with minimal latency—a technique rarely possible in traditional interrupt-driven architectures.

## 6. Instructions with Functional Differences

### BRK Instruction

| OP | LEN | CYC | MODE | FLAGS      | SYNTAX    |
|----|-----|-----|------|------------|-----------|
| 00 | 1   | 7   | zp   | ....01..   | BRK       |

**Flags Affected:** D (decimal), I (interrupt disable); also B (break) flag of pushed status

**65C02 Difference from 6502:**

The 65C02 clears the D (decimal) flag when BRK executes, whereas the 6502 does not affect D.

**Distinguishing BRK from IRQ:**

Both BRK and IRQ push the processor status (P) register onto the stack (after pushing the return address), but with a key difference:

- BRK sets bit 4 (B flag) of the pushed status to 1
- IRQ clears bit 4 (B flag) of the pushed status to 0

Interrupt handlers can distinguish the two by examining bit 4:

```assembly
PHX
TSX
PHA
INX
INX
LDA $100,X
AND #$10
BNE BREAK
BEQ IRQ
```

**Incorrect Approach:**
```assembly
PHA
PHP
PLA
AND #$10
BNE BREAK
BEQ IRQ
```

This method fails because it examines the current P register, not the value pushed by BRK/IRQ.

**65C02 Enhancement for Hardware Interrupts:**

The 65C02 also clears the D flag when IRQ, NMI, and RESET interrupts occur (after pushing the P register), whereas the 6502 leaves D unaffected (IRQ/NMI) or undefined (RESET).

**Advantage:** Interrupt handlers need not execute CLD to establish a known D flag state, saving two cycles—valuable in time-sensitive interrupt handling.

## 7. Instructions with Different Cycle Counts

### ASL LSR ROL ROR - abs,X Addressing Mode

| OP | LEN | CYC | MODE   | FLAGS      | SYNTAX        |
|----|-----|-----|--------|------------|---------------|
| 1E | 3   | 6 a | abs,X  | N.....Z.   | ASL $1234,X   |
| 5E | 3   | 6 a | abs,X  | N.....Z.   | LSR $1234,X   |
| 3E | 3   | 6 a | abs,X  | N.....Z.   | ROL $1234,X   |
| 7E | 3   | 6 a | abs,X  | N.....Z.   | ROR $1234,X   |

**Note a)**: 7 cycles if indexing across a page boundary

**65C02 vs. 6502 Timing:**

- **6502:** Always 7 cycles, regardless of page boundary crossing
- **65C02:** 6 cycles when no page boundary crossed; 7 cycles when crossing a page boundary

The instructions are otherwise functionally identical.

**Note on DEC/INC:** Unlike ASL, LSR, ROL, ROR, the instructions DEC abs,X and INC abs,X consistently require 7 cycles on both 6502 and 65C02, regardless of page boundary status. Some documentation incorrectly claims DEC/INC follow ASL/LSR/ROL/ROR timing; this is false.

## 8. Instructions with Functional and Cycle-Count Differences

### ADC SBC

| OP | LEN | CYC | MODE   | FLAGS      | SYNTAX          |
|----|-----|-----|--------|------------|-----------------|
| 69 | 2   | 2 a | imm    | NV....ZC   | ADC #$12        |
| 65 | 2   | 3 a | zp     | NV....ZC   | ADC $34         |
| 75 | 2   | 4 a | zp,X   | NV....ZC   | ADC $34,X       |
| 72 | 2   | 5 a | (zp)   | NV....ZC   | ADC ($34)       |
| 71 | 2   | 5 b | (zp),Y | NV....ZC   | ADC ($34),Y     |
| 61 | 2   | 6 a | (zp,X) | NV....ZC   | ADC ($34,X)     |
| 6D | 3   | 4 a | abs    | NV....ZC   | ADC $5678       |
| 7D | 3   | 4 b | abs,X  | NV....ZC   | ADC $5678,X     |
| 79 | 3   | 4 b | abs,Y  | NV....ZC   | ADC $5678,Y     |
| E9 | 2   | 2 a | imm    | NV....ZC   | SBC #$12        |
| E5 | 2   | 3 a | zp     | NV....ZC   | SBC $34         |
| F5 | 2   | 4 a | zp,X   | NV....ZC   | SBC $34,X       |
| F2 | 2   | 5 a | (zp)   | NV....ZC   | SBC ($34)       |
| F1 | 2   | 5 b | (zp),Y | NV....ZC   | SBC ($34),Y     |
| E1 | 2   | 6 a | (zp,X) | NV....ZC   | SBC ($34,X)     |
| ED | 3   | 4 a | abs    | NV....ZC   | SBC $5678       |
| FD | 3   | 4 b | abs,X  | NV....ZC   | SBC $5678,X     |
| F9 | 3   | 4 b | abs,Y  | NV....ZC   | SBC $5678,Y     |

**Note a)**: One additional cycle if D flag is set (decimal mode)
**Note b)**: One additional cycle if D flag is set (decimal mode); another additional cycle if indexing across a page boundary

**Binary Mode (D flag clear):**
ADC and SBC function identically on both 6502 and 65C02.

**Decimal Mode (D flag set):**

**Functional Differences:**

On the 65C02, the N, V, and Z flags produce valid results in decimal mode, unlike the 6502 where only the accumulator and carry results are reliable.

- **Z Flag:** Set when accumulator is $00; cleared for any other value (including invalid BCD with hex digits A-F)
- **N Flag:** Reflects bit 7 state; mathematically less meaningful for unsigned BCD but consistently reported
- **V Flag:** Produces the same result on 6502, 65C02, and 65816; useful primarily for diagnostic purposes
- **Accumulator Result:** Valid on both processors
- **Carry Result:** Valid on both processors

**Cycle Count Differences:**

ADC and SBC require one additional cycle in decimal mode on the 65C02:
- ADC #$01: 2 cycles (binary mode) → 3 cycles (decimal mode)
- SBC #$01: 2 cycles (binary mode) → 3 cycles (decimal mode)

This applies across all addressing modes.

**Combined Cycle Penalties:**

When multiple conditions apply, cycles accumulate. For example, with D flag set and X register at $FF:
- SBC $1234,X: 4 cycles (abs,X) + 1 (page boundary) + 1 (decimal) = 6 cycles

**Undocumented Behavior in Decimal Mode:**

The 6502 and 65C02 produce different accumulator results in decimal mode with invalid BCD:

```assembly
SED
SEC
LDA #$20
SBC #$0F
; 65C02 result: Accumulator = $0B
; 6502 result: Accumulator = $1B
```

### JMP (abs) - Indirect Jump

| OP | LEN | CYC | MODE | FLAGS      | SYNTAX      |
|----|-----|-----|------|------------|-------------|
| 6C | 3   | 6   | (abs) | ........   | JMP ($1234) |

**Bug Fix from 6502:**

The 6502 contained a documented bug in JMP (abs): when the low byte of the operand was $FF (e.g., JMP ($12FF)), the processor incorrectly fetched the high byte of the jump address from $1200 instead of $1300.

**Example:**
- Intended behavior: Low byte from $12FF, high byte from $1300
- 6502 behavior: Low byte from $12FF, high byte from $1200 (incorrect)

**65C02 Fix:**

The 65C02 corrects this bug, properly fetching both address bytes:
- Low byte from $12FF
- High byte from $1300 (correct)

**Cycle Count Impact:**

Due to the fix, JMP (abs) requires 6 cycles on the 65C02 versus 5 cycles on the 6502.

## 9. Unused Opcodes (Undocumented NOPs)

On the 6502, only 151 of 256 possible opcodes behaved as intended; the remaining 105 executed with unpredictable effects—some useful, others destructive (e.g., opcode $02 locked up the processor).

The 65C02 treats all unused opcodes as NOPs (no operation), preventing harmful side effects. However, some undocumented NOPs exhibit subtle behaviors worth understanding.

### Characteristics of 65C02 Undocumented NOPs

**Primary Behavior:** No flags, registers, or memory locations are modified (written).

**Secondary Behaviors:**

1. **Memory Reads:** Some undocumented NOPs perform memory reads beyond normal instruction fetching, risking unwanted I/O side effects
2. **Interrupt Latency:** One-cycle NOPs temporarily delay interrupt recognition; multiple consecutive one-cycle NOPs merge execution with the following instruction

### Undocumented NOP Table

### Applications of Undocumented NOPs

**Cycle-Precise Timing:**
One-cycle NOPs (opcodes $x3, $x7, $xB, $xF patterns) provide a smaller alternative to the standard two-cycle NOP ($EA) when exact cycle counts are critical.

```assembly
; Standard NOP is 2 cycles
NOP        ; 2 cycles

; One-cycle NOPs are 1 cycle (be cautious with interrupt latency)
DB $03     ; 1 cycle
DB $07     ; 1 cycle
DB $0B     ; 1 cycle
DB $0F     ; 1 cycle
```

**Cautionary Note:** Interrupt recognition can be delayed after one-cycle NOPs, particularly when multiple consecutive one-cycle NOPs execute. The processor effectively merges subsequent instruction execution, potentially deferring interrupt response.

**Conditional Skip Patterns:**

Technique: Skip a single byte unconditionally using a two-byte NOP:

```assembly
           ...    ; (preceding code)
           ...    ; (preceding code)
           DB  $42
INC_X:     INX
           LDA SOMELOCATION,X
           ...    ; etc
```

The $42 opcode (two-byte, two-cycle NOP) skips the following byte without affecting any flags or registers. Advantages over branch-based skips:

- Does not require flags to be in a specific state
- Preserves all flags and registers
- Requires only two cycles (optimal for multi-byte instructions)
- Compatible with 65816 (where $42 = WDM, a documented NOP)

**Two-Byte Skip Using Undocumented NOPs:**

```assembly
           ...    ; (preceding code)
           ...    ; (preceding code)
           DB  $DC
OVERRIDE:  LDA #$FF
           STA WHEREVER
           ...    ; etc
```

Using $DC (three-byte absolute-mode NOP) skips two bytes unconditionally. Alternative opcodes include $2C (BIT abs) and $CD (CMP abs), but these affect flags via memory reads. The $DC approach preserves flags while skipping:

- $DC uses absolute addressing; operand bytes ADL and ADH become the skip targets
- Performs a memory read (potentially problematic with I/O)

**Advantage of $DC/$FC over $2C/$CD:**
- Flags remain unmodified
- Still requires flag-safe skipping when I/O is not involved

**Drawback:** Both approaches perform a read at the operand address, risking unwanted I/O side effects.

**Prefix Byte Encoding:**

Advanced technique: Use one-cycle NOPs as prefix bytes to modify the behavior of the following instruction via external hardware:

```assembly
DB $03          ; Prefix: enable bank switching
JMP ROUTINE_A   ; Execute in alternate bank

DB $07          ; Prefix: enable another mode
LDA $1000       ; Execute with different memory mapping
```

The one-cycle NOP's bit pattern is easily detected via the SYNC signal, allowing external logic to respond immediately. The interrupt delay mechanism ensures the prefix and following instruction execute atomically—a critical property preventing interrupt-induced prefix loss.

**Multi-Cycle NOP Memory Access:**

Certain multi-cycle NOPs ($44, $54, $D4, $F4, $DC, $FC) perform address-mode-based memory reads. External hardware can latch address lines or data for banking operations:

```assembly
DB $DC        ; Read from absolute address specified by next two bytes
DW BANK_TABLE ; Address to latch
...           ; Actual code continues
```

This technique enables register loading without affecting the accumulator, X, or Y registers—impossible with traditional memory-mapped I/O on the 65C02.

**Comparison with 6502 Undocumented Opcodes:**

The 65C02's tidy treatment of unused opcodes contrasts sharply with the 6502's chaotic undocumented behavior. While the 6502 offered some useful undocumented instructions (e.g., undocumented ANE for certain bit operations), it also included destructive opcodes. The 65C02 eliminates this unpredictability while preserving exploitable behaviors through its systematic NOP approach.

## Summary

The 65C02 represents a significant refinement of the original 6502, introducing:

1. **Enhanced Addressing Modes:** New options for existing instructions, enabling more efficient code
2. **New Instructions:** Dedicated operations for common tasks (BRA, STZ, PHX/PHY, PLX/PLY, TSB, TRB)
3. **Manufacturer-Specific Extensions:** Bit manipulation (BBR, BBS, RMB, SMB) on Rockwell and WDC variants
4. **Power Management:** Sleep and halt instructions (WAI, STP) on WDC 65C02s
5. **Bug Fixes:** Corrected JMP (abs) indirect jump behavior and improved decimal-mode arithmetic
6. **Systematic Undocumented Behavior:** Reliable NOP treatment enabling creative hardware integration

These improvements make the 65C02 a more capable and programmer-friendly processor while maintaining backward compatibility with 6502 code.
